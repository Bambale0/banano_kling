# UX-анализ Telegram-бота Banano/Kling

Дата анализа: 2026-04-23

## 1. Что сейчас есть в боте

Бот уже выглядит как полноценная AI-студия внутри Telegram. Основные сценарии:

| Сценарий | Что делает | Где в коде |
|---|---|---|
| Главное меню | Дает вход в фото, видео, Motion Control, фото -> промпт, партнерку, оплату, поддержку и помощь | `bot/keyboards.py:18` |
| Создание видео | Text -> Video, Image + Text -> Video, Video + Text -> Video; модели Kling, Veo, Grok/Glow | `bot/keyboards.py:105`, `bot/handlers/generation.py:61` |
| Создание фото | Референсы до 14 фото, выбор модели, формата и количества | `bot/handlers/generation.py:93`, `bot/keyboards.py:367` |
| Motion Control | Фото персонажа + видео движения -> анимированный результат | `bot/handlers/generation.py:139` |
| Фото -> Промпт | Анализ изображения и выдача промпта | `bot/handlers/image_analyzer.py:17` |
| Пополнение | Пакеты бананов через CryptoBot | `bot/keyboards.py:461`, `bot/handlers/payments.py:38` |
| Партнерка | Реферальная ссылка, статистика, оферта, вывод | `bot/handlers/common.py:457` |
| AI-ассистент | Помощь с выбором модели/промпта/формата | `bot/handlers/common.py`, `bot/services/ai_assistant_service.py` |

## 2. Текущее главное меню

Так оно фактически собирается сейчас:

```text
🏠 Главное меню
Создавайте изображения, видео и эффекты в одном аккуратном потоке.

Что можно сделать
• Арты и иллюстрации по промпту
• Редактирование и стилизация фото
• Видео из текста, фото и референсов
• Motion Control и апгрейды результата

🍌 Баланс: 120 бананов

┌────────────────────┬────────────────────┐
│ 🎬 Создать видео   │ 🖼 Создать фото     │
├────────────────────┼────────────────────┤
│ 🎯 Motion Control  │ 📸 Фото=Промпт      │
├────────────────────┼────────────────────┤
│ 💼 Партнёрам       │ 💰 Пополнить        │
├────────────────────┼────────────────────┤
│ 🆘 Тех. поддержка  │ ❓ Помощь бота      │
└────────────────────┴────────────────────┘
```

Сильная сторона: все ключевые функции видны сразу.

Слабая сторона: меню не отвечает на главный вопрос новичка: "что мне выбрать под мою задачу?" Фото, видео, Motion Control и Фото=Промпт стоят на одном уровне, хотя это разные типы намерений: создать, улучшить, проанализировать, оплатить, заработать, получить помощь.

## 3. UX-проблемы и риски

### 3.1. Слишком много выбора на втором экране видео

Экран видео сразу показывает тип генерации, модели, форматы, длительности, Veo-настройки, цену и кнопку главного меню. Для опытного пользователя это быстро, но для нового пользователя когнитивная нагрузка высокая.

Особенно тяжело:

```text
Тип -> Модель -> Формат -> Длительность -> Veo/Grok доп. настройки -> Промпт
```

Проблема не в количестве возможностей, а в том, что все они видны одновременно.

### 3.2. Кнопка цены выглядит как действие

В `get_create_video_keyboard()` есть кнопка:

```text
💰 15🍌
```

Но ее `callback_data="back_main"`, то есть нажатие возвращает в главное меню. Пользователь может подумать, что это запуск/оплата/подтверждение, нажать и потерять контекст.

Лучше сделать цену текстом в сообщении или неактивной кнопкой через `callback_data="ignore"` с ответом "Это стоимость текущих настроек".

### 3.3. Фото-меню начинается с референсов, а не с намерения

Сейчас "Создать фото" сразу ведет в шаг "Референсы". Это хорошо для продвинутого edit/i2i сценария, но не идеально для простого "хочу картинку по тексту".

Для большинства новичков первый шаг должен быть:

```text
Что делаем?
[✨ С нуля по тексту]
[✏️ Изменить фото]
[🧩 По референсам]
```

### 3.4. Motion Control дублируется смыслом

Есть отдельная кнопка "Motion Control" в главном меню и есть "Видео + Текст -> Видео (Motion Control)" внутри видео. Это может путать: пользователь не понимает, это один и тот же режим или два разных.

Рекомендация: назвать единообразно:

```text
🎯 Motion Control
Фото + видео движения -> анимация
```

А внутри видео не писать "Видео + Текст -> Видео (Motion Control)", если это фактически Kling Glow/референсное движение.

### 3.5. Нет быстрых шаблонов результата

Боту очень не хватает "намерений":

```text
📱 Reels / Shorts
🛍 Карточка товара
👤 Аватар / портрет
🎞 Рекламный ролик
🧑‍🎨 Стилизация фото
```

Сейчас пользователь должен сам знать модель, формат, длительность и промпт. Хороший UX может сначала спросить "для чего результат", а потом выставить параметры.

### 3.6. Партнерка слишком рано в главном меню

"Партнёрам" полезно, но для обычного пользователя это вторичная функция. В главном меню она конкурирует с созданием контента. Если продукт больше про генерацию, партнерку лучше унести в "Еще" или нижний ряд.

### 3.7. В текстах местами не хватает переносов

В партнерском экране несколько строк склеиваются из-за отсутствия `\n` между фрагментами. В Telegram это будет выглядеть плотнее и менее аккуратно.

## 4. Рекомендуемая информационная архитектура

Я бы разделил меню на 4 смысловых блока:

```text
1. Создать
   - Фото
   - Видео
   - Motion Control

2. Улучшить / разобрать
   - Фото -> Промпт
   - Улучшить промпт
   - Повторить / вариации

3. Деньги и аккаунт
   - Баланс
   - Пополнить
   - История

4. Помощь и рост
   - AI-ассистент
   - Помощь
   - Партнерка
```

## 5. Вариант UX A: "Студия на главном экране"

Подходит, если аудитория уже понимает, что хочет создать.

```text
🏠 Banano AI Studio

🍌 Баланс: 120
Что создаем сегодня?

┌────────────────────┬────────────────────┐
│ 🖼 Фото             │ 🎬 Видео            │
├────────────────────┼────────────────────┤
│ 🎯 Motion Control  │ 📸 Фото -> Промпт   │
├────────────────────┼────────────────────┤
│ 💰 Баланс/оплата   │ 💬 AI-помощник      │
├────────────────────┴────────────────────┤
│ ⋯ Еще: партнерка, помощь, история        │
└─────────────────────────────────────────┘
```

Плюсы:

| Плюс | Почему важно |
|---|---|
| Меньше хаоса | 8 кнопок превращаются в 6 основных |
| AI-помощник виден сразу | Он может продавать правильный сценарий |
| Деньги объединены | "Баланс/оплата" понятнее, чем только "Пополнить" |

Минус: партнерка становится менее заметной.

## 6. Вариант UX B: "Сначала задача, потом модель"

Лучший вариант для конверсии новичков.

```text
🏠 Что нужно сделать?

┌─────────────────────────────────────────┐
│ ✨ Создать с нуля                       │
│ текст -> фото или видео                 │
├─────────────────────────────────────────┤
│ ✏️ Изменить фото                        │
│ стиль, фон, объект, образ               │
├─────────────────────────────────────────┤
│ 🎬 Оживить фото                         │
│ фото -> видео, Motion Control           │
├─────────────────────────────────────────┤
│ 📸 Разобрать фото в промпт              │
│ бесплатно, для повторения стиля         │
├─────────────────────────────────────────┤
│ 💬 Помочь выбрать                       │
└─────────────────────────────────────────┘

Нижний ряд:
┌──────────────┬──────────────┬──────────────┐
│ 🍌 Баланс    │ 💼 Партнерка │ ❓ Помощь    │
└──────────────┴──────────────┴──────────────┘
```

После нажатия "Создать с нуля":

```text
✨ Создать с нуля

┌────────────────────┬────────────────────┐
│ 🖼 Фото             │ 🎬 Видео            │
├────────────────────┼────────────────────┤
│ 📱 Для Reels       │ 🛍 Для товара       │
└────────────────────┴────────────────────┘
```

Плюсы:

| Плюс | Почему важно |
|---|---|
| Новичок выбирает задачу, а не модель | Меньше ошибок |
| Можно автоматически выставлять настройки | Формат, модель, длительность |
| Проще продавать дорогие модели | "Хочу качество" понятнее, чем "Veo 3.1 Quality" |

Минус: для опытных пользователей на один клик дольше. Это лечится кнопкой "⚡ Быстрый режим".

## 7. Вариант UX C: "Пресеты под соцсети"

Подходит, если бот продается создателям контента, маркетологам, SMM.

```text
🏠 Выберите формат результата

┌────────────────────┬────────────────────┐
│ 📱 Reels/TikTok    │ 🖼 Пост/баннер      │
├────────────────────┼────────────────────┤
│ 🎞 Рекламный ролик │ 🛍 Карточка товара  │
├────────────────────┼────────────────────┤
│ 👤 Портрет/аватар  │ 🧑‍🎨 Стилизация     │
├────────────────────┼────────────────────┤
│ ⚙️ Свои настройки  │ 📸 Фото -> Промпт   │
└────────────────────┴────────────────────┘

🍌 Баланс: 120     💰 Пополнить     💬 Помощник
```

Пример пресета:

```text
📱 Reels/TikTok

Автонастройки:
• Тип: видео
• Формат: 9:16
• Длительность: 5-8 сек
• Модель: Kling Std / Veo Fast

┌────────────────────┬────────────────────┐
│ 📝 По тексту        │ 🖼 По фото          │
├────────────────────┼────────────────────┤
│ 💎 Максимум качества│ ⚡ Быстро/дешевле   │
└────────────────────┴────────────────────┘
```

Плюсы:

| Плюс | Почему важно |
|---|---|
| Очень понятно коммерческой аудитории | Пользователь мыслит результатом |
| Легко делать upsell | "Максимум качества" вместо технических названий |
| Меньше промпт-ошибок | Можно показывать примеры под формат |

Минус: нужно аккуратно поддерживать пресеты и цены.

## 12. Микрокопирайтинг кнопок

Лучше заменить некоторые подписи:

| Сейчас | Лучше |
|---|---|
| `Фото=Промпт` | `📸 Фото -> Промпт` |
| `Пополнить` | `🍌 Баланс / Пополнить` |
| `Тех. поддержка` | `💬 Поддержка` |
| `Помощь бота` | `❓ Как пользоваться` |
| `Видео + Текст -> Видео (Motion Control)` | `🎞 Видео-референс` или `🎯 Motion Control` |
| `💰 15🍌` как кнопка | `Стоимость: 15🍌` в тексте |
| `Seed: auto` | `🎲 Seed: авто` |
| `Watermark: off` | `🏷 Водяной знак: выкл` |

## 13. Быстрые улучшения без большого рефакторинга

1. Убрать кнопку `💰 {cost}🍌` или сделать ее неактивной.
2. Переименовать главное меню в "Banano AI Studio" и сгруппировать кнопки по задачам.
3. Добавить кнопку `💬 AI-помощник` в главное меню, а не только в поддержку.
4. Добавить быстрый экран "Что делаем с фото?" перед референсами.
5. Спрятать Veo seed/watermark/resolution в "Расширенные настройки".
6. Добавить "▶️ Запустить" как явное действие после ввода промпта/подтверждения.
7. Сделать единый термин для Motion Control во всех местах.
8. Добавить пресеты `Reels`, `Портрет`, `Товар`, `Реклама`.
